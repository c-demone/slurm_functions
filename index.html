    pyslurm_decorators - Submit Python Functions to a SLURM Cluster

    pyslurm_decorators is a Python library that provides a set of useful decorators for submitting Python functions as jobs to a SLURM cluster. The hope is to allow certain compute-heavy functions to be offloaded to a SLURM cluster, while carrying out smaller work locally.

    The decorators are built on the Dask library, which was built as extension to Numpy arrays and Pandas Dataframes for managing large datasets. The dependency graph below shows hows pyslurm_decorators is dependent on the different components of dask including:

    1. dask: provides all of the basic functionality of dask including Arrays, DataFrames, and Bags as well as some non-distributed diagnostic tools
    2. distributed: where most of the code for running distributed workloads is found: this is where the distributed task scheduler lives along with distributed diagnostics
    3. dask_jobqueue: deployment of Dask distributed library on batch job queueing systems such as PBS, SLURM, or SGE.

    Daks is comparable to Apache Spark/PySpark, in that they both allow you to work with large datasets using familiar constructs such as DataFrames. However, Apache Spark is a Java/Scala project and ivolves starting workers as JVM processes whereas Dask can launch it's workers as batch processes through an appropriate batch scheduler, such as SLURM. While Spark is a full platform, including machine learning and data streaming in a adddition to data processing features, Dask is only built for data processing. In contrast to Dask, however, Apache Spark Arrays/DataFrames in PySpark are built as APIs for the Python Numpy/Pandas libraries rather than extensions to them. As a result, while Dask may not include machine learning libraries of it's own it does integrate well with most popular Python libraries such as Scikit-Learn or Keras. For data streaming Dask can be integrated with other Python libraries such as Streamz. An online slide presentation is available that looks into using the Streamz library with Dask for data streaming

    Since the decorators are built on Dask, they should in principal be applicable to both Dask and non-Dask workloads. In other words, you should be able to create functions that utilize Dask objects such as Arrays, DataFrames, or Bags and they should work equally as well as a function that return "Hello World!" as a string. This also means other Python libraries that are built on Dask can be utilized within a decorated function.

    üìù Note: Currently only works when running locally on a SLURM HPC (e.g. Edith)

    üìã Key Features

    • Provides @slurm decorator that will submit decorated function to a SLURM cluster as defined by the user

    • Allows working with Dask without having to iniate clusters manually

    • Enables off-loading of compute-heavy Python functions to a SLURM HPC, return results and pass on to other parts of your script that may be run locally.

    • Provides an in-depth view of your function's resource usage through the Dask dashboard that is initiated when calling your decorated fiunction.

    Note: This project is still under development.

    Quickstart

    Clone the repository and try out some examples

    üìç Move into the repo and load anaconda3/2021.05 (or your favourite version, as long as python >= 3.6)

    $ cd pyslurm_decorators
$ module load anaconda3/2021.05

    üìç Ensure the proxy is loaded and install all requirements for slurmjobs

    $ module load proxy
$ pip3 install --user -r requirements.txt
---> 100%

    üìç Once all requirements are installed, the proxy should be unloaded.

    $ module unload proxy

    üìç Then try some examples in the examples directory

    $ cd examples
